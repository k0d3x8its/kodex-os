# 📝 Phase I — Observe & Document

This is the first phase which is broken down into two Epochs. The first Epoch is all about searching and observation where the second Epoch is about documenting what is relevant and necessary from the observation.

---

## Epoch I — Search & Observe

The first epoch is pretty simple. This is where you search the world for interesting data/information that is relevant to what you want to learn, need to learn, or just want to know or remember. When you find something relevant or noteworthy you then move to Epoch II for documentation. If you are able to preview what you've found, then document the relevant information from the preview.

---

## Epoch II — Document

The second Epoch is where you document what you have found. If it can be written down on paper, then it is merely written down in the memo book — but only if it is noteworthy. A date when it was written should be stamped on the note (ex: `3/3/23`). If it is more than I can write, and is on a website, then I use [bit.ly](https://bit.ly) to document the website with a brief description of what the link provides.

<details>
<summary>Shorten URL Usage Requirements & Reasons</summary>

1. Documentation that requires more text than can fit in the memo book pages and/or the link needs to be referenced later to save from keeping the website in your browser — or any other reason you find relevant
2. Name the link to remind you of the information within so an understanding is registered in memory when glancing at the link. Encase the link within `< >` without the website main body; exclude `bit.ly/` (less to write). If not obvious: make the link as short as possible
3. Uppercase letters are underlined. This is optional; when I write I tend to write sloppy and use capital letters when I do not need to or shouldn't. Because it is merely field notes, then it does not matter until I need to create a shortened URL

</details>

</details>
